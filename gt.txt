  - Reset clicked_points = []
  - Tạo cửa sổ "Select Points" ở (700, 50), gắn mouse_cb
  - Vòng while chờ: waitKey(30) bắt phím, q để hủy, Enter để xác nhận
  - Trong lúc chờ, vẫn cập nhật polar
  - Trả về 2 tọa độ (x, y) của P1 và P2

map_lidar_to_image(bar_points, p1, p2)
  Ánh xạ điểm LiDAR thuộc thanh sang pixel trên ảnh bằng NỘI SUY TUYẾN TÍNH:
  - Tìm góc nhỏ nhất (min_a) và lớn nhất (max_a) trong thanh
  - Với mỗi điểm, tính tỷ lệ: t = (angle - min_a) / (max_a - min_a)
    → t nằm trong khoảng 0 đến 1
  - Nội suy pixel: px = P1.x + t × (P2.x - P1.x)
                    py = P1.y + t × (P2.y - P1.y)
    → Khi t=0 ra P1, t=1 ra P2, t=0.5 ra giữa P1-P2
  - Kết quả: mỗi điểm LiDAR có thêm pixel_x, pixel_y

  - Lưu ảnh kết quả calibration PNG
  - Lưu file JSON chứa: timestamp, P1, P2, dải góc, số điểm, mapped_points

calibrate(pipeline, align, scan_data, timestamp, polar_ax, polar_line, polar_fig, count)
  Điều phối toàn bộ quy trình calibration 1 lần:
  1. capture_frames() — chụp ảnh camera TRƯỚC (tránh timeout khi chọn điểm lâu)
  2. select_bar_points() — mở scatter, chọn 2 đầu thanh trên LiDAR
  3. select_image_points() — mở ảnh RGB, chọn 2 đầu thanh trên ảnh
  4. map_lidar_to_image() — nội suy tuyến tính mapping LiDAR → pixel
  5. show_result() — hiển thị kết quả
  6. save_all() — lưu ảnh + JSON

main_loop(lidar, writer, pipeline, align)
  Vòng lặp chính:
  - Tạo biểu đồ polar
  - Chạy thread LiDAR (daemon=True: tự tắt khi chương trình kết thúc)
  - Mỗi 0.2s cập nhật polar
  - plt.pause(0.05): chờ 50ms, xử lý sự kiện GUI
  - Khi nhấn 's': lấy scan mới nhất → chạy calibrate()
  - save_count đếm số lần calibration thành công

cleanup(lidar, csv_file, pipeline)
  Dọn dẹp khi thoát:
  - Tắt cờ lidar_running → thread LiDAR dừng
  - Chờ 0.5s cho thread kết thúc
  - Đóng file CSV, ngắt LiDAR, đóng biểu đồ, đóng cửa sổ ảnh, dừng camera

PHẦN CHẠY CHÍNH (__main__):

load_calibration_jsons(calibration_dir)
  Đọc toàn bộ file JSON calibration, gom thành 2 mảng numpy:
  - Tìm tất cả file khớp mẫu "calibration_*.json" bằng glob
  - Với mỗi file, đọc phần mapped_points
  - Chuyển tọa độ cực (angle, distance) sang Cartesian (x, z):
      x = distance × sin(angle)   → trục ngang (phải dương, trái âm)
      z = distance × cos(angle)   → trục sâu (phía trước dương)
  - Tại sao chuyển: LiDAR trả dạng cực, nhưng Homography cần tọa độ Descartes
  - Kết quả:
      lidar_pts: mảng Nx2, mỗi dòng [x, z] (mm)
      pixel_pts: mảng Nx2, mỗi dòng [pixel_x, pixel_y]

compute_homography(lidar_pts, pixel_pts)
  Tính ma trận Homography H (3×3):

  Homography là gì:
    Phép biến đổi phối cảnh 2D→2D. Với điểm LiDAR (x, z), tính pixel (u, v):
    [u', v', w'] = H × [x, z, 1]
    u = u'/w',  v = v'/w'

  cv2.findHomography(lidar_pts, pixel_pts, cv2.RANSAC, 5.0)
    - RANSAC: thuật toán loại điểm nhiễu (outlier). Thử nhiều tổ hợp 4 điểm,
      tìm H mà đa số điểm thỏa mãn (sai số < 5 pixel).
    - Trả về: H (3×3) và mask (mảng 0/1 chỉ điểm nào là inlier)

  Tính reprojection error (sai số chiếu ngược):
    - Thêm cột 1: [x, z] → [x, z, 1] (tọa độ homogeneous)
    - Nhân H: projected = H × [x, z, 1]^T
    - Chia w: pixel = projected[:2] / projected[2] (chuẩn hóa)
    - errors = khoảng cách Euclidean giữa pixel dự đoán và pixel thật
    - In mean, median, max, std

visualize_results(lidar_pts, pixel_pts, H, calibration_dir)
  Hiển thị ảnh so sánh:
  - Đọc ảnh calibration PNG mới nhất (hoặc tạo ảnh đen nếu không có)
  - Vẽ chấm XANH LÁ (5px): ground truth — pixel thật mà người dùng đã click
  - Vẽ chấm ĐỎ (3px): projected — pixel dự đoán bằng H × [x, z, 1]
  - Nếu 2 loại chấm gần nhau → calibration tốt, xa nhau → cần thêm dữ liệu

  1. Load tất cả file JSON → gom lidar_pts + pixel_pts
  2. Kiểm tra đủ ≥ 4 cặp điểm

  5. Hiển thị ảnh so sánh

normalize_angle(angle)

load_calibration()
  Đọc file calibration_result.npz:

  - Kiểm tra có key 'homography' trong file
  - Lấy ma trận H (3×3) và error

lidar_to_pixel(angle_deg, distance_mm, H)
  Chuyển 1 điểm LiDAR (góc, khoảng cách) → pixel (u, v) bằng Homography:

  Bước 1 — Polar → Cartesian:
    x = distance × sin(angle)   → trục ngang
    z = distance × cos(angle)   → trục sâu

  Bước 2 — Nhân Homography:
    pt = [x, z, 1]              → tọa độ homogeneous
    pixel = H × pt              → [u', v', w']

  Bước 3 — Chuẩn hóa:
    u = u' / w'                 → tọa độ pixel ngang
    v = v' / w'                 → tọa độ pixel dọc

  Kiểm tra w' ≈ 0 (phép chia không hợp lệ) → trả None.
  Trả về (u, v) dạng số nguyên.

dist_to_color(dist_mm, min_d=300, max_d=4000)
  Tính màu điểm theo khoảng cách (gradient xanh→vàng→đỏ):
  - ratio = (dist - 300) / (4000 - 300), giới hạn trong [0, 1]
  - ratio < 0.5 (gần, 0.3m→2.15m): giữ Green=255, tăng Red từ 0→255 → xanh lá dần sang vàng
  - ratio ≥ 0.5 (xa, 2.15m→4m): giữ Red=255, giảm Green từ 255→0 → vàng dần sang đỏ
  - Trả về (B, G, R) định dạng BGR cho OpenCV

lidar_thread()
  Thread LiDAR chạy ở background:
  - Kết nối LiDAR ngay trong thread (khác collect: lidar tạo trước rồi truyền vào)
  - Đọc liên tục iter_scans()
  - Lọc ±90°, distance > 0
  - Ghi vào latest_scan qua scan_lock
  - Khi cờ running=False hoặc lỗi → dừng, ngắt LiDAR

init_realsense()

main()
  Vòng lặp fusion real-time:

  2. Chạy thread LiDAR daemon ở background
  3. Khởi tạo camera RealSense

  4. Vòng lặp mỗi frame:
     a. Lấy ảnh color từ camera (wait_for_frames, align)
     b. Tạo bản sao ảnh để vẽ lên (frame = color_img.copy())
     c. Lấy bản sao scan LiDAR mới nhất (qua scan_lock)
     d. Duyệt từng điểm (angle, dist):

        - Kiểm tra pixel nằm trong ảnh: 0 ≤ u < 640, 0 ≤ v < 480
        - Tính màu bằng dist_to_color(dist)
        - Vẽ chấm tròn 4px tô đặc tại (u, v)

================================================================
CÔNG THỨC TOÁN HỌC CHÍNH
================================================================

1. Chuyển tọa độ cực → Descartes:
   x = distance × sin(angle)
   z = distance × cos(angle)

2. Nội suy tuyến tính (trong collect, mapping thanh):
   t = (angle - angle_min) / (angle_max - angle_min)     → t ∈ [0, 1]
